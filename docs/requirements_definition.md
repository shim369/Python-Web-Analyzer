# サイトリニューアル可否自動判定ツール 要件定義書

## 2. 機能要件

### 2.2. ファイル入出力機能

* **Excelインポート**:
* `B列（サイト名）`に純粋なドメイン名が記載されたExcelファイル（`.xlsx`）を受け取る。
* システムは解析時、自動的に `https://` などのプロトコルを補完してリクエストを行う。


* **Excelエクスポート**:
* 解析完了後、元の行順を完全に維持したまま、A列に実行日の日付（`M/D` 形式）、C列〜O列に解析・判定結果を書き込んだExcelファイルを出力する。



### 2.3. 各列のデータ取得・出力ロジック仕様

| 列 | 項目名 | データ型 | 処理・取得・判定ロジック |
| --- | --- | --- | --- |
| **A** | 日付 | 文字列 | **アプリ実行時の日付を `M/D` 形式（例: `7/14`）で出力。** |
| **B** | サイト名 | 文字列 | インポートされたドメイン名（そのまま維持）。 |
| **C** | 調査結果 | 文字列 | 画面指定の閾値に基づき判定。(1) `J列（使用CMS）`が空欄、かつ `I列` ≦ 【閾値1】 ➔ **◎** ／ (2) 上記以外、かつ `I列` ≦ 【閾値2】 ➔ **◯** ／ (3) 上記以外、かつ `I列` ≦ 【閾値3】 ➔ **△** ／ (4) いずれにも該当しない場合 ➔ **×** |
| **D** | SSLあり | 文字列 | トップページに `https://` でアクセスを試み、有効なSSL証明書が確認できれば **◯**、不可なら **×** または空欄。 |
| **E** | SSL常時 | 文字列 | 下層ページ等でもSSLが有効、かつ `http://` へのアクセスが `https://` へ自動リダイレクトされれば **◯**、そうでなければ **×**。 |
| **F** | 階層数 | 整数 | URLのディレクトリ構造（スラッシュ `/` の数）を基準とする最大階層数。 |
| **G** | svcmd | 文字列 | 常に**空欄**（NULL値）として出力する。 |
| **H** | 構成 | 文字列 | サイトのHTMLからグローバルナビゲーションを検出。メニューテキストを**改行区切り**で表示。 |
| **I** | ページ数 | 整数 | 同一ドメイン内のすべてのリンクを巡回し、総ページ数を集計。**10秒以内**に取得できなければ**空欄**とする。 |
| **J** | 使用CMS | 文字列 | メタタグ、ディレクトリ構造等からCMS（WordPressなど）を識別。 |
| **K** | 用途 | 文字列 | トップページのメタタグ `<meta name="description" content="...">` の文字列。 |
| **L** | 問合せ項目 | 文字列 | 問い合わせページ内の項目名を**改行区切り**で表示。見つからない場合は早期に空欄離脱。 |
| **M** | 不可の理由 | 文字列 | (1) `I列（ページ数）` ≧ 【閾値3】 ➔ 「**ページ数が多いため**」 ／ (2) 会員ログイン機能を検出 ➔ 「**会員ログイン機能があるため**」 ※併記あり |
| **N** | 備考 | 文字列 | 特殊なスクリプト等の検知時に出力。基本は空欄。 |
| **O** | 担当 | 文字列 | **アプリ実行時にユーザーが手動入力した「担当者名」**をそのまま出力。 |

## 3. 非機能要件

### 3.1. 性能・制限事項（スクレイピング制御）

* **タイムアウト管理**: 1ドメインあたりのクロールおよびページ数集計処理（H列）は最大10秒でタイムアウトとする。これにより、特定の大規模サイトや応答の遅いサイトによってツール全体の処理がスタックするのを防ぐ。
* **クローリングの安全性**: 対象サーバーへ過度な負荷を与えないよう、同一ドメイン内を巡回する際は適切なディレイ（間隔）を設けてリクエストを送信する。
* **Contactページの早期離脱**: 複雑なサイト構造において問い合わせページが発見できない場合、無限ループや過度なネスト探索を回避するため、一定の探索ステップで見つからなければ即座にスキップ（空欄）とする。

### 3.2. 処理方式

* **非同期キュー処理**: 複数ドメインが記載されたExcelファイルがアップロードされた場合、Webアプリケーションのメインプロセスとは別に、バックグラウンド（非同期スレッドやタスクキュー）で安全にスクレイピングを実行する。
* **進捗の可視化**: 処理中は画面側へ現在の処理件数（例: `25 / 100件 完了`）をリアルタイム、または定期的な自動更新（ポーリング）によって通知する。
